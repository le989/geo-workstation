# GEO-MVP-SIMPLIFY-1 页面内容与排版简洁化收口报告

## 1. 阶段目标

本阶段不新增功能，只做页面内容、排版和信息层级简洁化收口。

目标是减少说明过多、标签过多、卡片过多和测试提示过多的问题，让本地 MVP 更像清楚可用的 GEO 运营工具，而不是说明页或功能堆叠页。

## 3. 用户要求

- 页面名称不改。
- 不给每页新增“三句话”顶部说明。
- 页面内容要干净、简洁、明了。
- 排版要优化好。
- 不继续堆复杂功能。
- 最终阶段必须另开真实 AI 端到端验收，但本阶段不触发真实 AI。

## 4. 改前主要问题

- 功能增多后，页面有继续复杂化的趋势。
- Dashboard 快捷入口偏多，容易再次抢占主视线。
- 侧边栏主流程、复盘分析和辅助工具的层级不够清楚。
- 引用证据中心和竞品占位原因页的说明偏长，容易像概念说明页。
- 提示词库顶部同时存在资产概览和业务价值卡片，信息密度偏高。
- 帮助页首屏用数量统计强调说明条目，容易显得像大百科。

## 5. 收口策略

- 主流程靠前：Dashboard、提示词库、知识库、发布文章工作台、AI 模型覆盖记录。
- 复盘分析降级：引用证据中心、竞品占位原因、GEO 诊断、GEO 报表。
- 辅助功能靠后：AI 拓词、售后问答、指令库、统计、日志、用户、部门、设置。
- 页面说明减量：保留必要边界，去掉重复解释和英文 kicker。
- 卡片和标签减量：保留核心指标，降低辅助卡片密度。
- smoke / 轻量推断提示降噪：每页保留必要提示，不在模块内反复强调。

## 6. 修改清单

### Dashboard

- 将底部快捷入口从 8 个压缩为 5 个主动作。
- 保留补问法、补证据、补文章、复盘模型覆盖、查看证据链。
- 去掉顶部右侧重复的 smoke / 非正式结论提示。
- 本阶段只做入口减量和提示降噪，不新增图表或指标。

### Geo Prompts

- 顶部说明压缩为一句工具化描述。
- 业务价值概览从 4 张卡片改为一行轻量摘要标签。
- 提示词列表说明压缩为“用于判断处理优先级”。
- 保留问法类型、业务价值、购买阶段展示，不改保存或筛选逻辑。

### Knowledge Bases

- 本阶段只做页面走查，现有证据类型、可引用状态和资料管理入口保持不变。
- 未新增说明卡片，避免知识库继续概念化。

### Geo Content

- 本阶段只做页面走查，AI 引用友好检查、复制、发布稿和任务流程保持不变。
- 未触发生成、优化、复制或发布。

### Model Inclusion Records

- 本阶段只做页面走查，品牌提及、品牌推荐、未推荐原因复盘保持不变。
- 列表和展开详情未改，避免影响原有操作入口。

### Evidence Citations

- 去掉英文 kicker。
- 顶部说明压缩为“查看问法、证据、文章和模型覆盖之间是否衔接”。
- 缺口分布和问题证据链说明改为更短的行动导向文案。
- 概览卡高度和数字字号收紧，降低高级复盘页压力。

### Competitor Occupancy

- 去掉英文 kicker。
- 顶部说明压缩为“查看竞品出现、我方缺席和下一步补救方向”。
- 竞品分布、原因分布和复盘列表说明缩短。
- 概览卡高度和数字字号收紧，保留轻量识别 / 需确认边界。

### Geo Analysis

- 本阶段只做页面走查，诊断流程保持不变。
- 在导航中从主流程降到复盘分析，避免首屏主链路过重。

### Expansion

- 本阶段只做页面走查，规则拓词和 AI 拓词入口保持不变。
- 在导航中降为辅助工具，避免抢主流程。

### Help

- 顶部说明压缩为“查看常用流程、操作边界和风险提醒”。
- 右侧统计从“帮助板块 / 说明数量”改为“快速查找操作边界”。
- FAQ 中引用证据中心和竞品占位的解释缩短。
- 未删除帮助内容，只降低首屏百科感。

### Settings

- 本阶段只做页面走查，设置保存、Provider、公司和项目档案逻辑保持不变。

### Navigation / Layout

- 侧边栏分组调整为：
  - `GEO 主流程`
  - `复盘分析`
  - `辅助工具`
  - `帮助与交接`
- 页面名称未改。
- 没有删除任何页面入口。
