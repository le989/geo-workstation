# GEO-DASHBOARD-CLARITY-TRIM-1 Dashboard 信息层级瘦身收口报告

## 1. 阶段目标

本阶段不是新增功能，而是对 `/dashboard` 做减法、重排和降噪。

目标是把 Dashboard 从“所有模块同时展开”收口为“第一眼能判断当前 GEO 状态和下一步动作”的 AI 可见度驾驶舱。本阶段不新增数据库字段，不新增 API，不接真实 AI，不接外部平台，不做正式部署。

## 3. 改前问题

- 模块过多：KPI、判断卡、趋势图、模型对比、未推荐原因、标签云、行动清单、TOP5、最近动态、辅助统计和快捷入口全部展开，页面过长。
- smoke / 测试标签重复：KPI、趋势、模型对比、原因分布等位置重复出现测试说明，视觉噪声偏高。
- 首屏重点不够：用户需要自己从多个卡片中拼出“本轮结论”和“下一步动作”。
- 行动清单偏后：补问法、补证据、补文章、复盘模型没有在第一屏形成明确任务入口。
- 辅助统计和快捷入口抢视线：资产规模和快捷入口仍占据较明显位置，容易弱化 GEO 主判断。

## 4. 本阶段收口策略

- 首屏聚焦 `KPI + 本轮结论 + AI 可见度趋势`。
- 将原来的四张判断卡合并为一张 `本轮结论` 卡，集中展示当前判断、主要短板和下一步动作。
- 将 `下一步建议动作` 前移到本轮结论卡中，让用户一眼看到补问法、补证据、补文章、复盘模型。
- 第二屏承接模型推荐对比、未推荐原因分布、高频问法 / 场景词标签云。
- TOP5、最近动态、辅助统计和处理入口下沉为辅助信息。
- 保留图表化驾驶舱，不退回纯文字说明页。
- 顶部只保留一个全局 smoke / 本地测试版提示，图表内仅用小号轻提示说明测试示例。

## 5. 修改清单

### Dashboard 结构调整

- KPI 从 6 张收为 5 张：品牌推荐率、品牌提及率、官网引用率、竞品占位率、待补救问题。
- `未命中率` 不再作为 KPI 卡单独占位，保留在趋势图图例中展示。
- 原 `AI 有没有推荐我 / 哪些模型推荐 / 为什么没推荐 / 下一步补什么` 四张判断卡合并为 `本轮结论`。
- `本轮结论` 卡展示当前判断、主要短板和主要处理入口。
- 补问法、补证据、补文章、复盘模型动作前移到本轮结论区域。

### 测试标签降噪

- 顶部保留一个全局提示：`本地测试版 · smoke 数据 · 非正式结论`。
- KPI 卡不再逐个显示 `smoke 实数 / 前端汇总` 标签。
- 图表卡仅保留小号 `测试示例` 轻提示。

### 辅助模块降级

- 模型推荐对比、未推荐原因分布、标签云下移到诊断分析区。
- 未覆盖问题 TOP 5、知识库缺口 TOP 5、最近动态下移到 `辅助信息` 区。
- `第二层统计` 改为 `辅助统计`。
- `快捷入口` 改为 `处理入口` 并保留在底部。

### 帮助说明更新

- `/help` 中 Dashboard 说明改为优先看 `本轮结论` 和 `下一步动作`，再看趋势、模型对比和原因分布。
