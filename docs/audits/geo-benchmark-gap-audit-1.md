# GEO-BENCHMARK-GAP-AUDIT-1 对标查缺补漏报告

## 1. 阶段目标

本阶段用于对照 2026 年 GEO / AEO / AI Visibility 工作台和 AI Search 规则，检查当前本地测试版 GEO 工作站是否能支撑“AI 能检索到、理解、信任、引用、推荐”的运营闭环。

本阶段不是新增功能阶段，不做 official 数据库操作，不做真实 AI 调用，不做正式部署。审计重点是判断现有产品是否能让运营人员知道：

- 当前品牌在 AI 回答里的可见度如何。
- 为什么某些问法没有提及或推荐品牌。
- 下一步应该补问法、补证据、补文章，还是补发布渠道。
- 哪些能力已经具备，哪些能力只适合作为后续阶段推进。

## 3. 外部对标结论

### AI Visibility Dashboard

主流 AI Visibility 工具已经不只展示传统 SEO 指标，而是围绕 AI 回答里的品牌表现组织驾驶舱。常见指标包括品牌是否出现、推荐率、引用率、竞品占位、趋势和待处理动作。

当前工作站已有品牌提及率、品牌推荐率、官网引用率、未覆盖问题、待优化建议和模型覆盖记录，但 Dashboard 仍有示例趋势、示例分布和“待接入统计”区域。它已经是 GEO 工作台雏形，但还不是完整的 AI 可见度驾驶舱。

### Prompt Tracking

AI Visibility 工具通常把 prompt / 问法作为核心追踪对象，关注提示词是否有业务价值、属于哪个购买阶段、是否被品牌命中、竞品是否占位。

当前 `/geo-prompts` 和 `/expansion` 已新增“问法类型”，能从关键词管理推进到用户真实问法管理。但还缺“业务价值”和“购买阶段”这两层运营优先级。

### Source Citations

Bing AI Performance 类工具重点展示 AI 答案引用了哪些 URL、哪些页面被引用、哪些 grounding queries 触发引用，以及页面引用趋势。当前工作站已有官网引用状态、引用来源 JSON、知识库资料依据和证据类型，但缺一个跨页面的“引用证据中心”：把问题、模型回答、引用来源、缺少证据类型串起来。

### Brand Sentiment

外部工具通常会分析 AI 如何描述品牌，是正向推荐、中性提及、谨慎评价还是负面反馈。当前工作站记录了提及、推荐、竞品、原始回答，但还没有情绪 / 语气 / 正负向表达的轻量分类。

### Competitor Benchmark

竞品对标通常不只是“出现竞品”，还要回答“谁被推荐了、在哪类问法里赢了、它为什么赢、我们缺什么”。当前模型覆盖记录可以记录竞品出现和竞品名称，并通过“未推荐原因复盘”给出补救方向，但还没有竞品占位原因的独立视图。

### Content AEO

内容 AEO 通常关注内容是否清晰、结构化、可引用、有 FAQ、有证据、有结论。当前 `/geo-content` 已有 `AI 引用友好检查`，覆盖首段回答、FAQ、场景、参数、风险、品牌表达、结论句和资料依据，已经满足本地测试版的轻量内容自查目标。

### Grounding Queries / 被引用页面

Bing AI Performance 的方向是让运营知道哪些查询短语触发了引用、哪些页面被 AI 引用。当前工作站有提示词、模型覆盖记录、官网引用状态和内容资产，但尚未形成“问法 -> 被引用页面 / 来源 -> 缺口”的固定视图。

### 多模型 / 时间 / 问题类型对比

外部平台常见按模型、地区、时间、问题类型、客户旅程筛选表现。当前工作站已有模型、平台、入口、提示词类型、意图、检测时间等字段和筛选基础，但趋势仍有限，地区 / 购买阶段 / 业务价值维度尚未形成。

## 4. 当前 GEO 工作站已具备能力

当前工作站已经完成一条轻量 GEO 运营闭环：

```text
用户真实问法分类
→ 知识库证据类型
→ 发布文章 AI 引用友好检查
→ 模型覆盖未推荐原因复盘
```

已具备能力包括：

- `/geo-prompts`：提示词列表和详情展示“问法类型”，支持选型问题、参数问题、应用场景问题、品牌对比问题、故障排查问题、价格 / 采购问题、替代型号问题、行业方案问题。
- `/expansion`：拓词候选展示“问法类型”，规则拓词示例更偏真实自然问句。
- `/knowledge-bases`：资料卡片、表格和详情展示“证据类型”，支持产品参数、应用场景、选型建议、安装接线、故障排查、品牌介绍、案例 / 项目、禁用表达。
- `/geo-content`：发布文章详情抽屉展示 `AI 引用友好检查`，辅助判断文章是否便于 AI 摘取、理解、引用和推荐。
- `/model-inclusion-records`：展开详情展示 `未推荐原因复盘`，把未提及、未推荐、未引官网、竞品占位转成可能原因和补救方向。
- `/geo-reports`：已有总览、提示词覆盖、模型覆盖、内容覆盖、知识库覆盖和优化建议。
- `/help`：已有日常 GEO 运营流程、模块说明和高风险操作边界。
- 数据库备份安全链路：已完成 smoke 真实备份验证和 official 首次备份前 dry-run 检查，但本阶段未继续 official 操作。

## 5. 主要差距清单

### P0：阶段收口前必须关注

#### Dashboard 还不是完整 AI 可见度驾驶舱

- 问题描述：Dashboard 已有品牌提及率、推荐率、官网引用率和待优化建议，但趋势、模型占比、内容状态仍有示例展示，待审核资料和售后反馈统计仍是“接口未提供”。
- 影响：运营人员能看到概况，但还不能完全依赖 Dashboard 判断 AI 可见度趋势、引用来源变化和优先级。
- 修改难度：中
- 是否建议下一阶段做：建议。适合作为 `GEO-DASHBOARD-VISIBILITY-POLISH-1`，只做指标口径和空状态收口，不做报表大改。

#### “未推荐原因”到“补救任务”的连接仍是提示级

- 问题描述：模型覆盖记录能展示可能原因和下一步建议，但还没有把原因稳定回流到某个内容任务、知识库补证据任务或提示词维护动作。
- 影响：运营人员知道大方向，但还需要人工判断应该去哪一页补、补哪条资料、补哪篇文章。
- 修改难度：中
- 是否建议下一阶段做：建议。先做轻量入口和文案，不要直接做自动任务生成。

#### GEO 术语和优先级在页面间仍需要再统一

- 问题描述：问法类型、证据类型、引用友好度、未推荐原因已出现，但 Dashboard、帮助页、报表和页面空状态对“先做什么”的表达还不完全一致。
- 影响：新运营人员会看到很多模块，但可能仍不知道第一天应该先补问法、补证据还是补内容。
- 修改难度：低
- 是否建议下一阶段做：建议。可以在本地预发布 QA 阶段顺手收口文案。

#### 页面存在示例趋势 / 示例分布，正式使用前需要明显降噪

- 问题描述：Dashboard 明确标注了示例趋势和示例分布，这是安全的，但在正式使用时容易被误解为真实指标。
- 影响：可能误导运营复盘，尤其是首次培训或演示时。
- 修改难度：低
- 是否建议下一阶段做：建议。正式使用前改为“暂无趋势数据”或接入真实周期数据。

### P1：建议补，能明显提升实用性

#### Prompt 缺少业务价值分级

- 问题描述：提示词已有类型、意图、优先级和问法类型，但没有高意向 / 中意向 / 低意向，也没有“对线索价值”的提示。
- 影响：运营人员难以判断哪些问法先测、先写、先补资料。
- 修改难度：低到中
- 是否建议下一阶段做：建议，阶段名可用 `GEO-PROMPT-BUSINESS-VALUE-LITE-1`。

#### Prompt 缺少购买阶段

- 问题描述：当前无法区分认知、选型、对比、采购、售后等阶段。
- 影响：内容策略会偏平均，无法按客户旅程安排文章和知识库资料。
- 修改难度：低到中
- 是否建议下一阶段做：建议和业务价值分级一起做。

#### 缺少引用证据中心雏形

- 问题描述：知识库证据、内容资料依据、模型引用来源分散在不同页面。
- 影响：无法一眼看到某个问题的“模型回答引用了什么、没引用什么、缺哪类证据”。
- 修改难度：中
- 是否建议下一阶段做：建议，但先做只读轻量版，不改数据库。

#### 竞品占位原因还不够集中

- 问题描述：模型覆盖记录能记录竞品，但没有集中展示“竞品在哪些问法里赢、原因是什么、我们缺什么证据”。
- 影响：竞品分析仍要人工翻记录。
- 修改难度：中
- 是否建议下一阶段做：建议作为独立轻量阶段，不和 Dashboard 大改混在一起。

#### Brand Sentiment 仍缺位

- 问题描述：当前只记录是否提及 / 推荐，没有区分正向、中性、负向或谨慎表达。
- 影响：品牌被提及但语气不佳时，提及率可能掩盖实际风险。
- 修改难度：中
- 是否建议下一阶段做：暂不作为第一优先级，可先在模型覆盖记录里做人工轻量标记或只读推断审查。

### P2：后续增强，不建议当前做

#### Bing AI Performance / Search Console 数据接入

- 问题描述：当前没有接入 Bing AI Performance、Search Console 或真实引用查询数据。
- 影响：无法自动获得 grounding queries、被引用页面和真实 AI citation 趋势。
- 修改难度：高
- 是否建议下一阶段做：不建议当前做。等本地测试版稳定、部署和账号授权准备好后再评估。

#### 官网 URL 引用记录还不够结构化

- 问题描述：已有官网引用布尔值和引用来源 JSON，但缺少 URL 级引用汇总、页面维度趋势和页面质量提示。
- 影响：无法知道哪类官网页面更容易被 AI 引用。
- 修改难度：中到高
- 是否建议下一阶段做：暂缓。

#### 发布渠道效果回填缺失

- 问题描述：系统不记录内容发布到官网、公众号、B2B 平台后的效果回填。
- 影响：内容生产与外部渠道表现之间仍断开。
- 修改难度：中
- 是否建议下一阶段做：暂缓。先保持人工发布，不做自动发布。

#### 同一问题多次运行的波动记录不足

- 问题描述：模型覆盖记录支持检测时间和记录，但还没有“同一问题多次复测波动”的专门视图。
- 影响：难以判断优化动作是否真的带来持续改善。
- 修改难度：中
- 是否建议下一阶段做：暂缓。

### P3：当前明确不做

#### 自动监测频率提升

- 问题描述：后续可以自动周期检测，但当前本地测试版不适合扩大自动化。
- 影响：会带来 Provider、成本、稳定性和误触发风险。
- 修改难度：高
- 是否建议下一阶段做：不建议。

#### 自动发布

- 问题描述：当前系统明确人工发布，自动发布会引入渠道账号和合规风险。
- 影响：可能误发未复核内容。
- 修改难度：高
- 是否建议下一阶段做：不建议。

#### 定时任务 / 后台备份按钮 / 恢复工具

- 问题描述：这些属于运维自动化，不是当前 GEO 产品查缺补漏目标。
- 影响：扩大风险面，尤其会接近 official 和备份恢复边界。
- 修改难度：高
- 是否建议下一阶段做：不建议。

## 6. 页面级审计结果

### `/dashboard`

- 当前作用是否清楚：较清楚，能看到品牌提及率、推荐率、官网引用率、未覆盖问题、待处理事项和快捷入口。
- 对标 GEO/AEO 缺口：还不像完整 AI Visibility 驾驶舱，趋势和分布仍有示例性质，引用来源、竞品占位和内容补救动作没有集中成一个主视图。
- 是否建议本阶段小修：不建议。本阶段只写报告，避免继续零散 UI 小修。
- 后续功能建议：做 `GEO-DASHBOARD-VISIBILITY-POLISH-1`，把示例图降噪，强化 AI 可见度、引用来源和下一步动作。

### `/geo-prompts`

- 当前作用是否清楚：清楚，已从提示词列表升级为用户真实问法资产管理。
- 对标 GEO/AEO 缺口：缺少业务价值分级和购买阶段，不能直接区分哪些问法更接近采购或售后转化。

## 8. 下一步建议

### 第一优先级：`LOCAL-PRELAUNCH-QA-1：本地测试版全站查缺补漏`

推荐理由：当前 GEO 闭环已经具备，下一步不宜继续叠小功能，应先做本地测试版全站 QA，收口示例数据、空状态、按钮反馈、术语一致性和页面拥挤问题。

建议范围：

- 页面正常打开。
- 关键入口可见。
- 不出现 `undefined` / `null` / `NaN`。
- 空状态和错误态专业。
- 写入按钮边界清楚。
- 不触发真实 AI。
- 不写 official。

### 第二优先级：`GEO-DASHBOARD-VISIBILITY-POLISH-1：Dashboard AI 可见度驾驶舱收口`

推荐理由：Dashboard 是运营第一屏。当前已具备指标基础，但示例趋势、示例分布和待接入统计需要在正式使用前降噪。

建议只做：

- 替换或弱化示例趋势 / 示例分布。
- 强化 AI 可见度、引用来源、未覆盖问题和下一步动作。
- 不做报表大改。
- 不新增数据库字段。

### 第三优先级：`GEO-PROMPT-BUSINESS-VALUE-LITE-1：Prompt 业务价值分级`

推荐理由：问法类型已经完成，下一步最自然的是帮助运营判断哪些问题更接近线索、采购或售后风险。

建议只做：

- 前端轻量展示高意向 / 中意向 / 低意向。
- 前端轻量展示认知 / 选型 / 对比 / 采购 / 售后。
- 不新增数据库字段。
- 不触发真实 AI。

### 暂缓方向

- `GEO-EVIDENCE-CITATION-CENTER-LITE-1`：有价值，但应等本地 QA 和 Dashboard 收口后再做。
- `GEO-COMPETITOR-OCCUPANCY-REASON-LITE-1`：有价值，但不应和 Dashboard 收口混在一个阶段。
