# KB-1 企业知识库资料分类与手动录入

## 第一版范围

KB-1 将企业知识库从普通文件库升级为可治理的 GEO 资料底座。

- 资料记录支持标题、资料类型、资料主题、适用模块、来源说明、可信度、审核状态。
- 文件上传支持 `txt`、`md`、`csv`、Excel（`xls` / `xlsx`）和 Word（`docx`）。
- 手动录入资料会生成一条资料记录，并同步生成知识片段。
- 文件和手动资料默认 `reviewStatus=pending`，管理员可以设置 `approved` 或 `disabled`。
- 文件列表支持卡片 / 表格视图切换，并可按关键词、资料类型、资料主题、审核状态、可信度、适用模块和正式引用状态筛选。
- 片段内容继续支持编辑；`viewer` 仍然只读。
- `storagePath` 仍只保存在后端，不返回前端。

## 资料类型

资料类型挂在 `KnowledgeFile.materialType` 上，知识片段保留现有 `KnowledgeChunk.materialType` 字符串以兼容旧数据。

- `product_material`：产品资料
- `aftersales_material`：售后资料
- `company_trust_material`：公司可信信息
- `content_reference_material`：内容引用资料
- `internal_process_material`：内部制度 / 流程资料
- `customer_case_material`：客户案例资料

旧文件默认归入 `content_reference_material`。旧片段里的 `faq`、`product_info` 等历史值继续展示，不强制迁移。

## 资料主题

KB-INGEST-UX-1 新增 `KnowledgeFile.materialTopic`，用于表达比资料类型更细的内容主题。旧资料 `materialTopic` 可以为空，列表和详情需要兼容展示。

固定主题第一版包括：

- 公司新闻
- 活动资讯
- 资质证书
- 培训资料
- 行业动态
- 选型资料
- 故障排查
- 安装接线
- 应用案例
- 品牌介绍
- 产品参数
- 售后流程

公司新闻、活动资讯、资质证书、培训资料、行业动态等不新增为资料类型，统一作为资料主题管理。管理员自定义主题后置，本阶段只预留字段和固定选项。

## 目录、类型、主题、模块和审核状态的区别

- 目录：资料放在哪个知识库中，方便人工整理和检索。
- 资料类型：资料是什么大类，影响系统判断资料性质，例如产品资料、售后资料或公司可信信息。
- 资料主题：更细的内容主题，例如资质证书、行业动态、故障排查、安装接线。
- 适用模块：决定资料可以服务哪些功能，例如内部检索、GEO 内容生成、售后问答、GEO 分析。
- 审核状态：决定资料能不能作为正式引用依据。

## 审核状态

- `pending`：待审核，新上传和手动录入默认值。
- `approved`：已通过，后续售后问答检索只应使用该状态。
- `disabled`：已停用，后续检索不应使用。

`platform_admin` 和 `company_admin` 可以管理审核状态。`operator` 可以按原角色策略上传或手动录入资料，但不能直接把资料设为已通过或已停用。

正式引用场景已经收口审核状态和可信度规则：售后问答与 GEO 内容生成只使用 `approved` 且高 / 中可信资料；待审核、已停用或低可信资料仍可在知识库中查看维护，但不会作为正式引用依据。

## 列表管理

知识库详情的“资料文件”页签支持两种视图：

- 卡片视图：适合快速浏览资料标题、资料类型、资料主题、所属目录、审核状态、可信度、适用模块和正式引用状态。
- 表格视图：适合管理员批量管理资料状态、解析状态和操作入口。

筛选区支持关键词搜索、资料类型、资料主题、审核状态、可信度、适用模块和正式引用状态。正式引用状态由现有字段推导：`reviewStatus=approved` 且 `trustLevel!=low` 的资料显示为“正式可引用”；其他资料显示为“不正式引用”，并标明未审核、已停用或低可信等原因。

## 轻量入库与高级资料属性

知识库详情的“新增资料”页签采用“简单入库 + 高级资料属性折叠”的形式，降低普通员工入库门槛。

默认只展示核心字段：

- 手动录入：资料标题、资料类型、所属目录、正文内容。
- 文件上传：选择文件、资料类型、所属目录；资料标题可在高级属性中补充，不填时默认使用文件名。

高级资料属性默认折叠，包含：

- 资料主题
- 适用模块
- 审核状态
- 可信度
- 来源说明
- 售后资料可见部门，仅售后资料显示
- 产品线 / 产品目录和兼容标签

手动录入保存后创建 `sourceType=manual`、`fileType=manual` 的资料记录，并同步生成一个知识片段，可在片段列表继续编辑。

手动录入不写 `storagePath`，也不模拟真实上传文件。

## 文件上传元信息

文件上传入口在入库向导内，默认只需选择文件、资料类型和所属目录；如需细调，可在高级资料属性中填写：

- 资料标题
- 资料类型
- 资料主题
- 适用模块
- 来源说明
- 可信度
- 审核状态
- 售后资料可见部门

旧上传调用仍可只传文件；后端会补默认元信息并保持统一响应 `{ code, message, data }`。

中文文件名会在上传链路中做显示名规范化。常见 multipart latin1 误解码导致的中文乱码会尝试还原为 UTF-8；同时会清理路径分隔符和控制字符。规范化后的文件名仅作为资料显示名和默认标题，真实 `storagePath` 仍由后端生成并脱敏，不返回前端。

对于较大的 Word / docx 或文件名明显像综合资料包的文档，页面会给出“建议拆分”提示，但不阻断上传。综合文档建议按资料卡片拆分，例如：

- 公司介绍
- 资质证书
- 产品资料
- 售后资料
- 客户案例
- FAQ
- 行业知识

一条资料只解决一个主题，更利于售后问答、GEO 内容生成和内部查询准确引用。

## 智能默认值

资料类型切换时，前端会按资料类型自动填充默认适用模块、审核状态和可信度；用户手动调整后不强制覆盖。

- 售后资料：默认适用售后问答、内部检索；审核状态待审核；可信度中。
- 产品资料：默认适用售后问答、GEO 内容生成、内部检索；审核状态待审核；可信度中。
- 公司可信信息：默认适用 GEO 内容生成、内部检索；审核状态待审核；可信度高。
- 内容引用资料：默认适用 GEO 内容生成、内部检索；审核状态待审核；可信度中。
- 内部制度 / 流程资料：默认适用内部检索；审核状态待审核；可信度中。
- 客户案例资料：默认适用 GEO 内容生成、内部检索；审核状态待审核；可信度中。

非管理员入库时审核状态仍固定为待审核。

## AI 引用规则说明

当前页面只在高级属性和帮助说明中保留轻量引用规则，避免主入库流程过重：

- 售后问答只引用已通过且高 / 中可信的售后资料；找不到再引用已通过且高 / 中可信的产品资料。
- GEO 内容生成可引用已通过且高 / 中可信的产品资料、公司可信信息、内容引用资料、客户案例资料，以及行业动态 / 新闻活动主题资料。
- 内部查询可以查看待审核资料，但待审核资料必须标记为不建议作为正式依据。
- 低可信资料只用于内部查询，不给售后问答或 GEO 内容生成等 AI 正式输出场景引用。

`KB-CITATION-RULE-1` 已将正式引用规则收口到后端检索入口：售后问答和 GEO 内容生成只会使用已通过且高 / 中可信的资料；低可信资料仍保留在知识库列表和详情中，管理员可以继续查看和维护。

## 售后资料访问限制

第一版只对 `aftersales_material` 启用部门可见性限制。

- 非售后资料默认按原知识库可见性内部可见。
- 售后资料可配置 `allowedDepartmentIds`。
- `platform_admin` / `company_admin` 不受售后资料部门限制。
- `operator` / `viewer` 访问售后资料时，必须属于允许且启用中的部门；部门停用后不再授予售后资料访问权。
- 跨公司部门 ID 会被后端拒绝。
- 旧 `text-import` 兼容片段如果没有关联资料记录且自身标记为 `aftersales_material`，普通用户默认不可见，避免绕过 `allowedDepartmentIds`。
- 本限制不影响 DEPT-1 的模块入口权限；两者是叠加关系。

## 片段编辑

上传解析和手动录入生成的片段都可编辑正文内容。

- 更新片段不会改变 `companyId`、`knowledgeBaseId`、`fileId`。
- `viewer` 不允许编辑或删除片段。
- 售后资料下的片段列表会继承资料记录的部门可见性。
- `disabled` 资料默认不参与后续检索；片段是否可编辑仍按现有角色写权限和售后资料访问限制判断。

## 验收方式

写入验收只使用 `geo_workstation_crud_smoke`，不要使用 `geo_workstation_clean`。

建议命令：

```bash
DATABASE_URL="<geo_workstation_crud_smoke>" pnpm --filter @geo-workstation/api prisma:migrate:deploy
DATABASE_URL="<geo_workstation_crud_smoke>" pnpm --filter @geo-workstation/api exec vitest run test/knowledge-file-parser.service.spec.ts test/knowledge-files.service.spec.ts
DATABASE_URL="<geo_workstation_crud_smoke>" JWT_SECRET="<test-secret>" BYPASS_AUTH_FOR_TESTS=true pnpm --filter @geo-workstation/api exec vitest run test/knowledge-files.controller.spec.ts
pnpm --filter @geo-workstation/web typecheck
pnpm --filter @geo-workstation/web build
```

不要运行 `api test:auth`、`api test:users`、`api test:api`。
