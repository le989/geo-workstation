# GEO 工作站版本更新记录

这里记录当前主要版本的能力说明，方便内部演示、交接和复盘。以下版本名只作为说明记录；如果某些 tag 在本地不存在，不需要强行创建。

## internal-mvp-v0.2

主要能力：

- 完成内部演示版 MVP 的核心业务闭环。
- 覆盖 GEO 工作台、GEO 分析、提示词策略库、AI 拓词、企业知识库、指令库、内容生成、模型覆盖记录和 GEO 报表。
- 支持从诊断、策略、知识库、生产到追踪的基本流程。

适合用途：

- 内部演示。
- 小范围真实项目试跑。
- 给别人讲清楚 GEO 工作站的完整闭环。

注意事项：

- 不包含自动发布。
- 不包含真实外部 AI 自动检测。
- 不包含复杂审批流或多租户 SaaS。

## deployment-ready-v0.1

主要能力：

- 补充部署准备文档。
- 补充环境变量示例。
- 补充 PM2、Nginx 和 Docker Compose 生产示例。
- 补充上线前检查清单。

适合用途：

- 准备部署到内网测试服务器或单 VPS 前做检查。
- 给接手部署的人说明配置边界。

注意事项：

- 这是部署准备版本，不代表已经真实上线。
- 不提交真实 `.env`、数据库密码或 AI API Key。

## auth-ready-v0.1

主要能力：

- 增加登录页。
- 增加 JWT 登录态。
- 增加基础鉴权守卫。
- 前端显示当前用户、角色和退出登录入口。

适合用途：

- 内部账号访问。
- 防止业务页面在未登录状态下直接展示。

注意事项：

- 不包含开放注册。
- 不包含忘记密码。
- 不包含 OAuth。
- 不包含多租户和复杂菜单级权限。

## ai-provider-ready-v0.1

主要能力：

- 增加 OpenAI-compatible AI Provider 抽象。
- 保留默认 Mock Provider。
- 可通过后端环境变量配置 DeepSeek、硅基流动或其他兼容服务。
- AI 调用记录 Provider、模型、请求时间、消耗量和任务 ID。

适合用途：

- 在保持演示可用的前提下，按需切换真实 AI。
- 测试 AI 拓词和内容生成的真实效果。

注意事项：

- API Key 只允许放在后端私有 `.env` / `.env.production`。
- 前端不输入、不保存、不展示 API Key。
- 真实 AI 调用会消耗额度。

## generic-workstation-v0.1

主要能力：

- 增加项目档案。
- 支持跨行业通用模板体系。
- 系统不再绑定固定公司、固定产品线或固定行业。
- 适用于企业品牌、产品、服务、课程、门店、本地生活和个人品牌。

适合用途：

- 把同一套 GEO 工作站方法迁移到不同项目。
- 开始新项目时先配置品牌、客户、语气和表达边界。

注意事项：

- 项目档案只提供上下文。
- 具体事实仍以知识库、提示词和任务输入为准。

## content-qa-v0.1

主要能力：

- 增加内容质量检查。
- 增加发布优化版生成。
- 聚焦事实依据、风险表达和 GEO 结构。
- 检查未证实参数、协议、认证、夸张宣传和品牌表达风险。

适合用途：

- 内容发布前审核。
- 把生成稿调整成更可信、更清楚、更适合 AI 摘取的版本。

注意事项：

- 质量检查是辅助流程。
- 最终发布仍需要人工确认。

## content-format-v0.1

主要能力：

- 增加富文本发布稿排版。
- 支持复制富文本 / Markdown / 纯文本。
- 支持下载 HTML。
- 优化标题、段落、列表和 FAQ 结构。

适合用途：

- 把内容整理成可复制到官网、公众号、B2B 页面或销售资料的发布稿。

注意事项：

- 不会自动发布到外部平台。
- 不会自动覆盖原始内容项。

## ui-polish-v0.1

主要能力：

- 优化前端 UI 和体验。
- 改善页面布局、表格、筛选、空状态、错误提示和操作说明。
- 让系统更适合长期自用、内部演示和交接。

适合用途：

- 日常运营。
- 给别人演示完整流程。
- 降低新用户理解成本。

注意事项：

- 该版本不新增后端业务能力。
- 仍保持后台工具的克制风格，不做营销落地页。
