# 内部演示指南

## 项目一句话定位

GEO 工作站是一套面向生成式 AI 搜索/问答场景的营销运营系统，用来建设品牌诊断、提示词策略、企业知识资产、GEO 内容生成、模型覆盖记录和优化复盘闭环。

它不是普通 CMS、普通文件柜或普通内容后台。演示时要始终围绕：

`GEO 诊断 -> 提示词策略 -> 企业知识库 -> GEO 内容生成 -> 效果记录 -> 优化建议`

## 当前 MVP 能演示什么

- 从 Dashboard 查看 GEO 资产建设进度和待优化方向。
- 创建 GEO 分析任务，运行 Mock 分析，查看品牌提及、推荐、官网引用、竞品、内容缺口、知识库缺口和提示词建议。
- 将分析建议转入提示词策略库。
- 管理训练词、蒸馏词、品牌词、场景词。
- 用规则、默认 Mock 或 `openai_compatible` 生成候选提示词，并人工勾选保存。
- 创建企业 GEO 知识库，文本导入或上传 txt/md/csv，解析为知识片段。
- 创建 GEO 内容指令模板。
- 选择提示词、知识库和指令模板创建默认 Mock 或 `openai_compatible` 内容任务。
- 人工录入或导入模型覆盖记录。
- 查看 GEO 报表和优化建议。
- 使用内部登录页进入系统，并从顶部栏退出登录。

## 推荐演示顺序

### 0. 打开登录页

页面：`/login`

操作：

1. 使用 seed 创建的默认管理员或内部账号登录。
2. 登录成功后进入 Dashboard。

要讲什么价值：

- 当前内部演示版已有最小登录保护，不再是完全开放后台。
- 登录态使用 JWT Bearer，适合内网试用的基础访问控制。
- 这不是复杂账号系统，暂不包含注册、忘记密码、OAuth 或多租户权限。

### 1. 打开 Dashboard

页面：`/dashboard`

要讲什么价值：

- 这是 GEO 运营首页，不是普通后台首页。
- 它把提示词资产、知识库资产、内容产出、模型覆盖记录和优化建议放在一处。
- 运营人员可以判断下一步应该补词、补资料、补内容还是补检测。

### 2. 打开 GEO 分析，创建并运行 Mock 分析任务

页面：`/geo-analysis`

操作：

1. 点击“新建分析任务”。
2. 输入品牌名称、官网、产品线、核心训练词和目标模型。
3. 创建后点击“运行 Mock 分析”。
4. 打开详情查看 summary、contentGaps、knowledgeGaps、promptSuggestions 和 modelResults。

要讲什么价值：

- GEO 分析关注的是 AI 回答里是否提到品牌、是否推荐品牌、是否引用官网、竞品是否出现。
- 第一版是 Mock 分析，不访问真实网站，不调用真实外部 AI 平台，也不是 SEO 扫描器。
- 分析结果的目的不是给一份报告，而是驱动后续补提示词、补知识库和补内容。

### 3. 将分析建议转入提示词策略库

页面：`/geo-analysis` 详情抽屉

操作：

1. 勾选 promptSuggestions。
2. 设置提示词类型、用户意图、优先级和追踪开关。
3. 点击“转入提示词库”。

要讲什么价值：

- 分析产生的问题会沉淀为 GEO 提示词资产。
- 重复提示词会被跳过，不会污染策略库。
- 转入后的提示词可用于内容生成和模型覆盖记录。

### 4. 打开提示词策略库，查看提示词资产

页面：`/geo-prompts`

要讲什么价值：

- 训练词、蒸馏词、品牌词、场景词是 GEO 策略的核心资产。
- 页面支持搜索、筛选、新增、编辑、软删除、批量导入和 CSV 导出。
- 提示词不是普通关键词，它代表用户会如何向 AI 提问。

### 5. 打开 AI 拓词，生成候选词并勾选保存

页面：`/expansion`

操作：

1. 使用手动组合生成候选词，或使用 AI 拓词表单。演示时建议保持默认 `mock`。
2. 查看重复标记。
3. 勾选非重复候选词保存到提示词库。

要讲什么价值：

- 拓词结果不会自动入库，必须先进入候选词，再由人工选择保存。
- 默认 AI 拓词是 Mock Provider；自用真实流程可选择 `openai_compatible`，但候选词仍不会自动入库。
- 真实 AI 会消耗接口额度，API Key 只在后端环境变量配置，前端不展示或保存。

### 6. 打开企业知识库，创建知识库并导入资料

页面：`/knowledge-bases`

操作：

1. 创建一个产品线知识库。
2. 使用文本导入创建知识片段。
3. 可上传 txt/md/csv 文件并查看解析状态。

要讲什么价值：

- 知识库不是普通文件柜，而是 AI 内容生成和 AI 可引用信息的事实底座。
- 第一版支持 txt/md/csv 基础解析，不支持 PDF、Word、Excel、URL 抓取、RAG 或向量数据库。

### 7. 打开指令库，创建 GEO 内容指令模板

页面：`/instruction-templates`

要讲什么价值：

- 指令模板沉淀 GEO 内容生产方法。
- 它服务于 AI 问答素材、选型指南、FAQ、国产替代、应用方案、产品对比等内容类型。
- 指令库不是普通 prompt 收藏夹。

### 8. 打开 GEO 内容生成，选择提示词 + 知识库 + 指令模板生成内容

页面：`/content-tasks`

操作：

1. 创建内容任务。
2. 选择 GEO 提示词、知识库和指令模板。
3. 查看生成的 content_items。
4. 编辑内容项或导出 Markdown。

要讲什么价值：

- 内容生成的目标是提升某些提示词下的品牌可见度和推荐概率。
- 演示建议使用默认 Mock 生成器；自用真实流程可选择 `openai_compatible`。
- 真实 AI 内容生成会消耗接口额度，并会把必要的提示词、知识片段摘要和指令发送给后端配置的模型。
- 任务和内容项是真实入库的，可编辑、软删除和导出。

### 9. 打开模型覆盖记录，录入模型表现

页面：`/model-inclusion-records`

要讲什么价值：

- 模型覆盖记录是 GEO 效果复盘依据。
- 第一版用人工录入或批量导入替代自动检测。
- 重点记录品牌是否被提及、是否被推荐、推荐位置、是否引用官网和竞品出现情况。

### 10. 打开 GEO 报表

页面：`/reports`

要讲什么价值：

- 报表不是普通后台统计，而是为了指导下一步优化。
- 可以查看提示词覆盖、模型覆盖、内容覆盖、知识库覆盖和优化建议。
- 优化建议会指出哪些高优先级提示词缺记录、缺内容、缺知识库资料。

## Mock 能力

- GEO 分析：Mock 分析器，不调用真实外部 AI，不访问真实网站。
- AI 拓词生成：默认 Mock Provider，可选 `openai_compatible`；演示建议默认 Mock。
- GEO 内容生成：默认 Mock 内容生成器，可选 `openai_compatible`；真实 AI 结果仍需人工审核后使用。
- 登录与退出：JWT Bearer 登录态、路由保护、当前用户展示和退出登录。

## 真实入库能力

- GEO 分析任务和模型结果入库。
- 提示词策略库新增、编辑、软删除、批量导入、CSV 导出。
- AI 拓词候选保存到提示词库。
- 企业知识库、知识文件、知识片段入库。
- 指令模板入库。
- 内容任务和内容项入库。
- 模型覆盖记录入库。
- 报表实时读取数据库统计。

## 暂未实现

- 真实外部 AI 检测。
- 外部 AI 平台自动检测和定时任务。
- 开放注册、忘记密码、短信/邮箱验证码、OAuth、多租户和复杂菜单级权限。
- 自动发布到外部媒体。
- 真实服务器部署执行。
- 多用户协作和审批。
- PDF/Word 报告导出。
- URL 抓取、整站采集、RAG 和向量数据库。

## 演示时常见问题

### 如果后端未启动怎么办

登录页或业务页面会显示“后端未连接”“登录失败”或“加载失败”，但前端不应白屏。按以下顺序启动：

```bash
docker compose up -d postgres
pnpm dev:api
pnpm dev:web
```

再访问：

```text
http://localhost:5173/dashboard
```

### 如果无法登录怎么办

- 确认已执行 `pnpm prisma:migrate` 和 `pnpm prisma:seed`。
- 确认 seed 前已经设置正确的 `DEFAULT_ADMIN_EMAIL` 和 `DEFAULT_ADMIN_PASSWORD`。
- 确认 API 的 `JWT_SECRET` 已配置，且 API 重启后没有误用另一个密钥。
- 开发环境可参考 `.env.example` 中的占位账号密码；共享部署必须使用私有强密码。

### 如果页面没有数据怎么办

先执行 seed：

```bash
pnpm prisma:seed
```

也可以运行后端 MVP 冒烟脚本快速创建一批演示数据：

```bash
pnpm dev:api
pnpm smoke:api
```

`pnpm smoke:api` 会先登录默认管理员。如果演示环境使用了自定义管理员账号，可以执行：

```bash
SMOKE_AUTH_EMAIL=admin@example.com SMOKE_AUTH_PASSWORD=replace_me pnpm smoke:api
```

### 如果 smoke:api 失败怎么办

- 确认 PostgreSQL 已启动。
- 确认后端在 `http://localhost:3000`。
- 确认 `.env` 中 `DATABASE_URL` 指向本地 PostgreSQL。
- 如果 API 使用其他端口，执行 `API_BASE_URL=http://localhost:3001 pnpm smoke:api`。

### 如果文件上传失败怎么办

第一版企业知识库只支持 `.txt`、`.md`、`.csv`，文件大小建议不超过 10MB。PDF、Word、Excel 和 URL 抓取不在当前 MVP 范围。
