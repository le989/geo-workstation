# 跨行业通用 GEO 模板体系

本文档定义 GEO 工作站的通用模板设计方式。模板不按固定行业包写死，而按“用户在 AI 中的决策场景”组织。行业可以作为示例，但不能成为系统固定结构或数据库枚举。

## 为什么不能把旧表达当成全行业通用模板

“厂家推荐”“故障排查”“国产替代”“选型指南”等表达更偏工业品、B2B 制造业或供应链采购场景。它们在教育、餐饮、本地生活、快消、个人品牌、服务咨询、课程报名、门店到店等项目中不一定自然适用。

通用 GEO 工作台应从以下问题出发：

- 用户会在 AI 里怎么问？
- 用户处在什么决策阶段？
- 用户需要判断什么风险、成本、适配度、信任度或下一步动作？
- 哪些事实必须来自知识库、目标提示词或用户输入？

因此模板应基于用户决策场景，而不是固定行业模板、固定产品词或固定企业表达。

## 通用模板设计原则

推荐组合方式：

`全局通用质量规则 + 用户决策场景模板 + 项目档案 + 知识库事实资料 + 用户输入`

不推荐：

`固定行业模板 + 固定产品词 + 固定企业表达`

各层职责：

- 项目档案定义“这个项目是谁、服务谁、怎么表达”。
- 知识库定义“有哪些可验证事实”。
- 通用模板定义“怎么组织内容”。
- 提示词定义“用户会怎么问 AI”。
- 内容生成把这四者组合起来。

## 通用模板类型

以下模板是用户决策场景模板，不是行业固定模板。

### 1. 通用需求决策指南

- 适用场景：用户想判断某个产品、服务、课程、门店、个人品牌、解决方案或项目是否适合自己。
- 适合问题：某产品怎么选；某服务适合什么人；某课程是否适合孩子；某门店适合什么场景。
- 内容结构：用户问题；适用条件；不适用或需确认条件；判断逻辑；资料准备清单；FAQ 总结。
- 禁止事项：不替用户直接做最终选择；不编造参数、价格、效果、认证或案例。
- 示例问题：某方案是否适合我的预算；某课程适合零基础吗；某门店适合亲子聚餐吗。以上仅为示例，不代表系统固定行业。

### 2. 通用问题诊断与改善建议

- 适用场景：用户遇到效果不好、体验不佳、故障、学习困难、服务不匹配或到店顾虑。
- 适合问题：使用中遇到问题怎么办；效果不稳定怎么办；学习进度慢怎么办；服务体验不符合预期怎么办。
- 内容结构：问题表现；可能原因；排查顺序；改善建议；需要补充的信息；何时联系供给方。
- 禁止事项：不建议违规操作；不直接推荐更贵方案；不把某类能力当成万能解决手段。
- 示例问题：某设备检测异常怎么办；某课程学不会怎么办；某服务交付不符合预期怎么办。以上仅为示例。

### 3. 通用 FAQ 与顾虑解答

- 适用场景：用户在咨询、购买、报名、到店、合作前有常见疑问。
- 适合问题：适合谁；有什么限制；需要准备什么；费用或效果怎么确认；是否值得信任。
- 内容结构：问题列表；简洁回答；判断条件；风险提醒；下一步确认项。
- 禁止事项：不承诺结果；不绕开限制条件；不编造政策、价格、库存或资格。
- 示例问题：报名之前要准备什么；到店前要预约吗；合作前要确认哪些资料。以上仅为示例。

### 4. 通用场景解决方案

- 适用场景：用户已经有具体场景，需要找到匹配方案。
- 适合问题：某场景适合什么方案；某人群适合什么服务；某需求该如何组合资源。
- 内容结构：场景描述；核心问题；方案思路；适用条件；实施步骤；边界与注意事项；FAQ。
- 禁止事项：不把适用于某场景写成所有场景都适用；不编造实施效果。
- 示例问题：团建适合什么门店；企业培训怎么设计；某场景用什么产品。以上仅为示例。

### 5. 通用对比与替代方案

- 适用场景：用户在多个产品、服务、课程、门店、品牌或方案之间比较。
- 适合问题：A 和 B 怎么比较；是否可以替代；哪种更适合当前场景。
- 内容结构：比较维度；适用条件；限制条件；替代边界；选择建议；资料确认清单。
- 禁止事项：不贬低竞品；不写“一定替代”；不编造性能、价格、案例或效果。
- 示例问题：某方案和另一种方案怎么比较；某品牌是否能替代另一品牌；线上课和线下课怎么选。以上仅为示例。

### 6. 通用信任建立与品牌证明

- 适用场景：用户想判断某个供给方、服务方、品牌、门店、机构或个人品牌是否可信。
- 适合问题：某品牌是否靠谱；某服务方是否专业；某个人品牌是否可信。
- 内容结构：品牌定位；可验证资料；适合人群；不适合或需确认情况；咨询前准备。
- 禁止事项：不写“行业领先”“最佳选择”“一定有效”等夸张表达；不编造资质和案例。
- 示例问题：某机构是否可信；某门店适合商务宴请吗；某顾问是否适合长期合作。以上仅为示例。

### 7. 通用行动前准备清单

- 适用场景：用户准备咨询、购买、报名、到店、合作、试用或实施前，需要准备信息。
- 适合问题：咨询前要准备什么；购买前要确认什么；到店前要注意什么；合作前要提供什么资料。
- 内容结构：目标确认；预算/时间/场景；必要资料；风险点；沟通清单；下一步动作。
- 禁止事项：不制造焦虑；不承诺准备后必然成功；不要求用户提供不必要的敏感信息。
- 示例问题：报名课程前要准备什么；咨询服务前要说明哪些情况；采购前要确认哪些条件。以上仅为示例。

### 8. 通用案例复盘与结果证明

- 适用场景：用户需要通过真实案例、过程复盘或结果证明理解项目能力。
- 适合问题：有没有类似案例；结果如何验证；过程怎么推进；适用边界是什么。
- 内容结构：背景；目标；过程；结果；可复用经验；限制条件；下一步建议。
- 禁止事项：只能使用知识库明确提供的案例和结果；不编造客户名称、数据、收益或背书。
- 示例问题：某服务有没有类似案例；某课程学员结果如何看；某方案上线后怎么复盘。以上仅为示例。

## 行业适配方式

行业适配层由用户按项目自由创建，不作为系统固定菜单或枚举。

示例：

- 工业品可以把“问题诊断”细化为安装、接线、检测异常、选型边界。
- 教育项目可以把“问题诊断”细化为学习问题、提分路径、家长顾虑。
- 餐饮或本地生活可以把“场景解决方案”细化为聚餐、团建、亲子、约会、商务宴请。
- 个人品牌可以把“信任建立”细化为专业背景、方法论、案例复盘、咨询前准备。
- 快消产品可以把“需求决策”细化为适用人群、使用场景、成分/功能边界。

这些仅为示例，不代表系统固定行业，也不应写成数据库枚举或固定导航。

## 推荐系统使用逻辑

1. 在 `/settings` 配置项目档案，定义项目是谁、服务谁、表达语气和禁止表达。
2. 在知识库中录入可验证事实，包括产品、服务、课程、门店、个人品牌、案例、FAQ、资质或解决方案。
3. 在提示词策略库中维护用户会怎么问 AI。
4. 在指令库中使用通用决策场景模板，必要时按项目自由改写。
5. 内容生成时组合项目档案、知识库、提示词和模板，生成可编辑的 GEO 内容资产。
