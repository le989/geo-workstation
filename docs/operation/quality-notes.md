## 2. 哪些提示词生成效果好

通常更容易得到可用内容的提示词：

- 带明确用户意图：例如“某产品或服务怎么选”。
- 带应用场景：例如“某场景适合什么方案”。
- 带比较任务：例如“某方案和另一种方案怎么比较”。
- 带供给方 / 服务方选择意图：例如“某服务方是否值得信任”。

## 3. 哪些提示词生成效果差

常见问题：

- 训练词过短，缺少场景和用户意图。
- 品牌词没有配套知识库资料，导致内容空泛。
- 供给方 / 服务方推荐类提示词缺少资质、案例、服务边界或事实支撑。
- 对比与替代类提示词缺少对比维度和禁用规则。

## 4. 知识库资料不足会导致什么问题

| 资料缺口     | 生成表现                               | 修复建议                                      |
| ------------ | -------------------------------------- | --------------------------------------------- |
| 缺关键事实   | 内容只有泛泛决策建议，无法支撑 AI 引用 | 补产品 / 服务 / 课程 / 门店的可验证事实和边界 |
| 缺应用场景   | 内容不能回答具体业务问题               | 补典型使用、咨询、购买、到店或合作场景资料    |
| 缺 FAQ       | 内容缺少问答式摘要                     | 补常见问题和标准答案                          |
| 缺案例或资质 | 品牌可信度表达弱                       | 补可公开案例、行业经验、认证或服务说明        |
| 缺对比资料   | 对比与替代内容容易空泛                 | 补对比维度和禁止夸大规则                      |

## 5. 指令模板如何影响输出

观察重点：

- 是否要求只使用知识库可验证事实。
- 是否要求输出可被 AI 摘取的问答式总结。
- 是否给出内容结构：用户问题、判断逻辑、方案说明、注意事项。
- 是否明确禁用编造参数、资质、客户案例。
- 是否针对不同内容类型使用不同模板。

## 6. 下一步 Prompt / 指令优化方向

优先优化：

1. AI 拓词 prompt：增加“真实用户会如何向 AI 提问”的约束，减少泛关键词。
2. 内容生成 prompt：进一步约束 JSON 输出，减少非标准结构。
3. 需求决策指南模板：固定判断维度、事实边界、适用场景和注意事项。
4. FAQ 模板：要求每个答案包含事实依据和可摘取短句。
5. 应用方案模板：强调场景、问题、方案、安装注意事项和可验证资料。

## 7. 真实 AI 输出审核原则

- 不直接发布未经人工审核的内容。
- 删除或改写未经知识库支撑的事实。
- 对参数、认证、客户案例、竞品比较保持谨慎。
- 优先发布能够补齐高优先级提示词缺口的内容。
- 将好的输出反向沉淀到指令库。
