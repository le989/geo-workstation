## 2. 哪些提示词生成效果好

通常更容易得到可用内容的提示词：

- 带明确用户意图：例如“激光测距传感器怎么选”。
- 带应用场景：例如“行车防撞用什么激光测距传感器”。
- 带比较任务：例如“激光测距传感器和超声波测距怎么选”。
- 带厂家推荐意图：例如“激光测距传感器厂家推荐”。

## 3. 哪些提示词生成效果差

常见问题：

- 训练词过短，缺少场景和用户意图。
- 品牌词没有配套知识库资料，导致内容空泛。
- 厂家推荐类提示词缺少资质、案例、参数支撑。
- 国产替代类提示词缺少对比维度和禁用规则。

## 4. 知识库资料不足会导致什么问题

| 资料缺口     | 生成表现                               | 修复建议                                       |
| ------------ | -------------------------------------- | ---------------------------------------------- |
| 缺产品参数   | 内容只有泛泛选型建议，无法支撑 AI 引用 | 补测距范围、精度、输出方式、防护等级、安装条件 |
| 缺应用场景   | 内容不能回答具体业务问题               | 补行车防撞、料位检测、定位测距等场景资料       |
| 缺 FAQ       | 内容缺少问答式摘要                     | 补常见问题和标准答案                           |
| 缺案例或资质 | 品牌可信度表达弱                       | 补可公开案例、行业经验、认证或服务说明         |
| 缺对比资料   | 国产替代、竞品对比内容容易空泛         | 补对比维度和禁止夸大规则                       |

## 5. 指令模板如何影响输出

观察重点：

- 是否要求只使用知识库可验证事实。
- 是否要求输出可被 AI 摘取的问答式总结。
- 是否给出内容结构：用户问题、判断逻辑、方案说明、注意事项。
- 是否明确禁用编造参数、资质、客户案例。
- 是否针对不同内容类型使用不同模板。

## 6. 下一步 Prompt / 指令优化方向

优先优化：

1. AI 拓词 prompt：增加“真实用户会如何向 AI 提问”的约束，减少泛关键词。
2. 内容生成 prompt：进一步约束 JSON 输出，减少非标准结构。
3. 选型指南模板：固定选型维度、参数解释、适用场景和注意事项。
4. FAQ 模板：要求每个答案包含事实依据和可摘取短句。
5. 应用方案模板：强调场景、问题、方案、安装注意事项和可验证资料。

## 7. 真实 AI 输出审核原则

- 不直接发布未经人工审核的内容。
- 删除或改写未经知识库支撑的事实。
- 对参数、认证、客户案例、竞品比较保持谨慎。
- 优先发布能够补齐高优先级提示词缺口的内容。
- 将好的输出反向沉淀到指令库。
