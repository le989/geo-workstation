# 发布稿富文本排版与复制

本文档说明内容项在质量检查和发布优化版之后，如何生成更接近发布状态的富文本稿件。

该能力只负责排版、预览、复制和下载，不负责自动发布，也不连接任何内容平台账号。

## 能力目标

发布稿排版用于解决三个问题：

- 将原文或发布优化版转换成可复制到内容平台编辑器的富文本。
- 同时提供 HTML、Markdown 和纯文本，便于不同发布场景使用。
- 保留 GEO 友好的结构，例如标题、小标题、列表、FAQ、资料确认提醒和结尾行动建议。

它不会新增事实，不会自动覆盖原 `content_item`，也不会写入发布记录。

## 支持风格

第一版支持四种发布风格：

| 风格          | 适用场景                                     |
| ------------- | -------------------------------------------- |
| 通用发布稿    | 适合多数内容平台复制粘贴。                   |
| 官网文章      | 适合官网 CMS、SEO 站、技术文章或产品知识页。 |
| 知乎 / 百家号 | 适合问答式、小标题清晰、段落较短的平台内容。 |
| 公众号草稿    | 适合粘贴到公众号编辑器后人工预览。           |

不同平台会过滤部分样式，因此富文本复制后仍需要人工预览。

## 输出格式

接口和前端会同时生成：

- `html`：可用于富文本复制或下载 HTML。
- `markdown`：适合继续编辑、存档或导入支持 Markdown 的编辑器。
- `plainText`：用于平台不支持富文本时的兜底复制。
- `copyTips`：提示适合粘贴到哪些平台，以及需要注意的预览事项。

## 富文本结构

富文本会尽量保留原文结构，并使用基础 HTML 标签：

- `h1`、`h2`、`h3`
- `p`
- `ul`、`ol`、`li`
- `strong`
- `blockquote`
- `hr`
- 少量 `article` 容器和内联样式

不会使用：

- `script`
- `iframe`
- 外链 CSS
- 复杂动画
- 平台账号接口

## 使用方式

在 `/geo-content` 的内容任务详情中：

`/content-tasks` 作为历史兼容入口保留，指向同一页面。

1. 先生成或打开一个内容项。
2. 可选：执行“质量检查”。
3. 可选：生成“发布优化版”。
4. 在“发布稿排版”区域选择：
   - 内容项
   - 稿件来源：原文或发布优化版
   - 发布风格
   - 是否包含 GEO 发布提示和发布前确认
5. 点击“生成富文本发布稿”。
6. 根据需要使用：
   - 复制富文本
   - 复制 Markdown
   - 复制纯文本
   - 下载 HTML
   - 下载 Markdown

## 复制富文本

前端会优先使用 Clipboard API 写入：

- `text/html`
- `text/plain`

如果浏览器不支持富文本复制，会降级为复制纯文本，并给出提示。

## 人工预览原因

即使富文本已经生成，发布前仍需人工预览：

- 微信公众号、知乎、百家号、官网 CMS 等平台会各自过滤样式。
- 标题层级、列表缩进、引用框可能被平台重置。
- 原文中的事实、参数、认证、价格、交期和案例仍需要人工确认。
- 品牌表达和行动建议需要根据具体发布渠道微调。

## 边界

当前阶段不做：

- 自动发布。
- 发布记录模块。
- 平台账号管理。
- 微信公众号 / 知乎 / 百家号 / 小红书 API。
- Word / PDF 导出。
- 富文本版本管理。
- 自动覆盖原内容项。
- 持久化发布稿。

发布稿排版只是把“原文或发布优化版”转换为更方便复制和人工发布的格式。
