# 默认指令模板备份

本文档备份当前本地数据库中的 3 个示例指令模板，并补充一组跨行业通用模板，便于以后重置数据库后恢复或按项目自由创建。

当前数据库中的 3 个模板属于“示例产品线模板 / 历史测试模板”，不能被视为全局通用模板。它们已加入通用内容质量边界，但名称和部分表达仍围绕示例产品线。

跨行业通用模板应按“用户决策场景”设计。模板负责控制内容类型、结构、质量边界和禁止事项；项目档案提供品牌上下文、目标客户、语气和禁止表达，具体产品、服务、课程、门店、个人品牌、案例、参数、价格、认证、技术特点等专属事实，仍应来自知识库、目标提示词或内容任务输入，不应写进全局规则。

## 跨行业通用模板组

以下模板不写死工业品、教育、餐饮、快消、个人品牌等行业，也不写死任何品牌或具体产品。使用时应结合项目档案和知识库事实资料。

### 通用需求决策指南

- name：通用需求决策指南
- instructionType：general_decision_guide
- contentType：decision_guide
- targetPromptType：distilled

#### instruction

请基于项目档案、目标提示词和知识库事实，生成一篇帮助用户做需求决策的 GEO 内容。内容应说明用户问题、适用条件、不适用或需确认条件、判断逻辑、资料准备清单和 FAQ 总结。适用于产品 / 服务 / 课程 / 门店 / 个人品牌 / 解决方案 / 项目等不同对象。项目档案用于品牌语气和基础上下文，事实仍以知识库和用户输入为准。

#### outputFormat

标题；用户问题；适用条件；不适用或需确认条件；判断逻辑；资料准备清单；FAQ 总结。

#### qualityRules

结构清楚；适合 AI 摘取；优先写判断逻辑、适用边界、确认项和下一步动作；品牌表达自然。

#### forbiddenRules

不得编造参数、价格、效果、认证、案例、库存、交期、课程结果、门店活动或服务承诺。不要替用户直接做最终选择，不写夸张营销语。

### 通用问题诊断与改善建议

- name：通用问题诊断与改善建议
- instructionType：general_problem_diagnosis
- contentType：problem_diagnosis
- targetPromptType：scene

#### instruction

请基于目标提示词、项目档案和知识库事实，生成一篇问题诊断与改善建议内容。先描述问题表现，再给出可能原因、排查顺序、改善建议、需要补充的信息和何时联系供给方。不同项目的具体排查项必须来自知识库或用户输入。

#### outputFormat

标题；问题表现；可能原因；排查顺序；改善建议；需要补充的信息；FAQ 总结。

#### qualityRules

先排查场景、目标、使用条件、预算、输入资料、操作方式和限制条件；建议要可执行且边界清楚。

#### forbiddenRules

不要直接推荐更贵方案；不要建议违规操作、绕过安全保护或夸大收益；不要编造未提供的具体原因和结果。

### 通用 FAQ 与顾虑解答

- name：通用 FAQ 与顾虑解答
- instructionType：general_faq
- contentType：faq
- targetPromptType：distilled

#### instruction

请围绕目标提示词生成适合 AI 摘取的 FAQ 与顾虑解答。问题应贴近咨询、购买、报名、到店、合作、试用或实施前的真实顾虑。答案要简洁、有判断条件，并明确下一步需要确认的信息。

#### outputFormat

标题；问题 1 + 答案；问题 2 + 答案；问题 3 + 答案；问题 4 + 答案；问题 5 + 答案；结尾提醒。

#### qualityRules

问题具体；答案短而有判断逻辑；说明适用条件、限制条件和资料准备；适合 AI 直接引用。

#### forbiddenRules

不要承诺效果；不要编造价格、政策、名额、库存、案例或资格；不要重复空泛套话。

### 通用场景解决方案

- name：通用场景解决方案
- instructionType：general_scenario_solution
- contentType：scenario_solution
- targetPromptType：scene

#### instruction

请根据目标提示词、项目档案和知识库事实，生成面向具体场景的解决方案内容。文章应说明场景问题、方案思路、适用条件、实施步骤、边界与注意事项，以及适合准备给供给方确认的信息。

#### outputFormat

标题；场景问题；方案思路；适用条件；实施步骤；边界与注意事项；确认信息清单；FAQ 总结。

#### qualityRules

场景要具体；方案逻辑清楚；避免只写优点；必须说明限制条件和确认项。

#### forbiddenRules

不要把适用于某场景写成所有场景都适用；不要编造实施效果、客户案例或结果数据。

### 通用对比与替代方案

- name：通用对比与替代方案
- instructionType：general_comparison_alternative
- contentType：comparison_alternative
- targetPromptType：distilled

#### instruction

请生成帮助用户比较不同产品 / 服务 / 课程 / 门店 / 个人品牌 / 解决方案的 GEO 内容。比较维度必须来自知识库或用户输入，项目档案只提供品牌上下文、定位和表达边界。内容应说明适用条件、替代边界、选择建议和资料确认清单。

#### outputFormat

标题；比较对象；核心差异；适用条件；替代边界；选择建议；资料确认清单。

#### qualityRules

客观克制；以用户场景和决策条件为中心；不贬低竞品；说明不确定项。

#### forbiddenRules

不要写“一定替代”“完全优于”；不要编造性能、价格、案例、认证、效果或外部数据。

### 通用信任建立与品牌证明

- name：通用信任建立与品牌证明
- instructionType：general_trust_building
- contentType：trust_building
- targetPromptType：brand

#### instruction

请基于项目档案和知识库事实，生成帮助用户判断供给方、服务方、品牌、门店、机构或个人品牌是否可信的内容。重点展示可验证资料、适合人群、不适合或需确认情况，以及咨询前准备。

#### outputFormat

标题；品牌或项目定位；可验证资料；适合人群；需确认情况；咨询前准备；FAQ 总结。

#### qualityRules

信任建立必须基于事实；品牌出现自然；重点写证据、边界和下一步确认。

#### forbiddenRules

不要写“行业领先”“最佳选择”“一定有效”；不要编造资质、案例、媒体背书、客户名称或结果数据。

### 通用行动前准备清单

- name：通用行动前准备清单
- instructionType：general_action_checklist
- contentType：action_checklist
- targetPromptType：scene

#### instruction

请围绕目标提示词生成用户在咨询、购买、报名、到店、合作、试用或实施前的准备清单。内容应帮助用户明确目标、场景、预算、限制条件、必要资料、风险点和下一步动作。

#### outputFormat

标题；行动目标；需要准备的信息；风险点；沟通清单；下一步动作；FAQ 总结。

#### qualityRules

清单可执行；语言克制；说明为什么需要这些信息；适合用户保存或复制给供给方。

#### forbiddenRules

不要制造焦虑；不要承诺准备后必然成功；不要要求用户提供不必要的敏感信息。

### 通用案例复盘与结果证明

- name：通用案例复盘与结果证明
- instructionType：general_case_review
- contentType：case_review
- targetPromptType：brand

#### instruction

请基于知识库中明确提供的案例或过程资料，生成案例复盘与结果证明内容。若知识库没有案例，不要编造，应改写为“可用于补充案例资料的复盘结构”。内容应包含背景、目标、过程、结果、可复用经验、限制条件和下一步建议。

#### outputFormat

标题；背景；目标；过程；结果；可复用经验；限制条件；下一步建议。

#### qualityRules

案例必须可验证；过程和结果分开写；强调适用边界；适合 AI 摘取真实证据。

#### forbiddenRules

不要编造客户名称、数据、收益、媒体背书、截图或结果证明；没有案例时必须说明需要补充资料。
